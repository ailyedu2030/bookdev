## 第一章 系统概述

### 1.1 项目背景

本项目旨在构建一个基于AI多智能体协同的教材编写系统，目标规模为**20万汉字**（约100万Tokens），服务于商学院企业管理、市场营销、人力资源管理等专业的教材开发。

**核心挑战**：
- **规模复杂性**：20万字教材需多轮迭代，涉及大纲生成、内容编写、审核修订的完整流程
- **质量一致性**：需保证AI生成内容的专业性、准确性和风格一致性
- **知识规范性**：教材内容需符合中国出版行业规范，满足CIP数据申报要求
- **成本可控性**：需在合理Token预算内完成，避免上下文膨胀导致的额外消耗

### 1.2 系统目标

| 目标维度 | 具体指标 |
|----------|----------|
| **规模目标** | 支持20万汉字（约100万Tokens）规模教材编写 |
| **质量目标** | 术语一致率≥95%、知识准确率≥98%、格式规范率≥99% |
| **效率目标** | 大纲生成<30分钟/章、内容生成<2小时/节、审核<30分钟/节 |
| **成本目标** | Token消耗≤150万/册（按MiniMax-M2.7 ¥2/1M tokens） |
| **合规目标** | 重大选题备案率100%、CIP数据完整率100%、原创性相似度<15% |

### 1.3 设计原则

本系统设计遵循以下核心原则，融合系统工程方法论与AI工程最佳实践：

1. **分层解耦**：将系统划分为规划层、执行层、认知层、工具层，各层职责明确
2. **状态驱动**：以知识图谱为状态核心，通过边结构维护章节间的逻辑关联
3. **过程可观测**：引入Temporal作为流程引擎，确保每个环节可追溯、可审查
4. **预算友好**：核心上下文60K + 弹性材料40K的双层架构，控制Token消耗
5. **合规内建**：将出版规范嵌入工作流节点，而非事后检查

## 第二章 技术架构设计

### 2.1 技术栈总览

| 层级 | 技术选型 | 核心职责 |
|------|----------|----------|
| **大模型** | MiniMax-M2.7 (200K context, ¥2/1M tokens) | 内容生成、理解推理、多轮对话 |
| **流程引擎** | Temporal + Redis Streams | 工作流编排、状态管理、断点续写 |
| **知识存储** | Neo4j + PostgreSQL | 知识图谱存储、结构化数据持久化 |
| **向量检索** | Redis (内置向量索引) | 语义检索、RAG上下文获取 |
| **缓存层** | Redis Cluster | Token预算管理、热点数据缓存 |
| **消息队列** | Kafka | 异步事件分发、日志聚合 |
| **前端** | React + Ant Design Pro | 编写界面、进度监控、结果展示 |
| **后端** | Python/FastAPI | API服务、业务逻辑、LLM封装 |

### 2.2 核心架构决策

#### 决策1：流程引擎选型

**问题**：教材编写是长时间跨度（数小时到数天）的工作流，需支持断点续写、错误恢复、并发编写。

**备选方案对比**：

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| LangChain Agents | 生态成熟、集成度高 | 长时间运行不稳定、调试困难 | 快速原型、POC |
| Temporal | 状态持久化好、断点续写原生支持、活动历史完整 | 学习曲线陡、运维成本高 | 生产级长时间工作流 |
| Kafka Streams | 高吞吐、事件驱动 | 状态管理弱、需手动处理断点 | 实时数据管道 |
| Redis Streams | 轻量级、低延迟 | 缺乏工作流原语 | 简单任务队列 |
| **Temporal + Redis Streams** | 互补性强：Tempora管工作流，Redis管实时事件 | 架构复杂 | **本章采用** |

**最终决策**：
- **Temporal**作为主流程引擎：负责章节编写工作流、审核流程、发布流程的编排
- **Redis Streams**作为实时事件总线：负责Token消耗监控、进度推送、实时协作事件
- **Kafka**作为日志聚合与异步任务队列：负责日志收集、审计事件、批量数据处理

#### 决策2：Token预算架构

**问题**：MiniMax-M2.7有200K上下文限制，20万字教材编写需多轮迭代，Token消耗必须精确管控。

**Token预算分配**（修订版）：

| 层级 | 预算 | 说明 |
|------|------|------|
| **L0 核心上下文** | 60K Tokens | 全局逻辑链（大纲架构、术语表、风格指南、角色设定） |
| **L1 弹性材料** | 40K Tokens | 当前章节的参考内容、上下文材料、RAG检索结果 |
| **L2 章节状态** | ~5K Tokens/章 | 章节元数据（进度、版本、审核状态） |
| **L3 输出缓冲** | 10K Tokens | 响应正文生成空间 |
| **总计** | **~115K Tokens/章节** | 预留15%余量应对上下文膨胀 |

**位置引用机制改为内容寻址**：

原方案使用位置基址（`token[0:100]`）存在上下文重放时位置偏移问题。

**新方案：内容寻址哈希**

```
内容寻址哈希 = SHA-256(内容片段文本)
引用格式 = {hash: "abc123...", offset: 0, length: 500}
```

优势：
- 位置无关：重放后引用仍然有效
- 自动去重：相同内容产生相同哈希
- 可验证：接收方可通过哈希验证内容完整性

#### 决策3：知识图谱设计

**问题**：如何维护教材的章节结构、知识点关联、术语体系？

**图谱设计**：

```
节点类型：
- Chapter（章）：id, title, order, status, version
- Section（节）：id, title, order, status, wordCount
- Subsection（节以下）：id, title, order, content
- Concept（概念）：id, name, definition, domain
- Term（术语）：id, term, definition, synonyms, domain

边类型：
- CONTAINS：章-节包含关系
- FOLLOWS：章节先后顺序（逻辑链）
- DEFINES：章节定义的概念
- USES：章节引用的术语
- SIMILAR_TO：概念相似关系（用于RAG排序）
- REFERENCES：章节间引用关系
```

```
┌─────────────────────────────────────────────────────────────────┐
│                      模块交互协议                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Agent A                    Agent B                             │
│    │                           │                                │
│    │──── Async Message ────────►│  (via Redis Pub/Sub)          │
│    │                           │                                │
│    │──── Workflow Signal ──────►│  (via Temporal)                │
│    │                           │                                │
│    │◄─── Context Request ──────│                                │
│    │──── Context Response ────►│  (via Redis, content-hash)     │
│    │                           │                                │
│    │──── Knowledge Update ─────►│  (via Neo4j Write)            │
│    │                           │                                │
└─────────────────────────────────────────────────────────────────┘
```

**通信原语**：
- **Async Message**：不阻塞的异步消息，用于进度同步、事件通知
- **Workflow Signal**：工作流信号，用于触发下游任务、确认完成
- **Context Request/Response**：上下文读写，基于内容寻址
- **Knowledge Update**：知识图谱更新，维护因果一致性

## 第三章 知识图谱架构

### 3.1 图谱设计理念

知识图谱是系统的**状态核心**，采用实体-关系-属性的基本结构，参考《专家讨论-知识图谱.md》的建模方法：

> 知识图谱应反映教材的认知结构，而非简单的章节目录。章节是容器，知识点是内容，关系是逻辑。

**核心设计原则**：
1. **章节即节点**：每个Chapter/Section对应图谱节点
2. **知识即属性**：概念、术语作为节点属性或独立节点
3. **逻辑即边**：章节间的先后、包含、引用关系通过边维护
4. **状态即时间**：通过版本节点维护状态变更历史

### 3.2 节点设计

| 节点类型 | 属性 | 说明 |
|----------|------|------|
| **Chapter** | id, title, order, status∈{待编写,编写中,已审核,已定稿}, wordCount, version, createdAt, updatedAt | 章节点 |
| **Section** | id, title, order, status, wordCount, parentChapterId, version, createdAt, updatedAt | 节节点 |
| **Subsection** | id, title, order, status, content, parentSectionId | 小节节点 |
| **Concept** | id, name, definition, domain, relatedTerms[], sourceChapterId | 概念节点 |
| **Term** | id, term, definition, synonyms[], domain, firstDefinedAt | 术语节点 |
| **Author** | id, name, role, assignedChapters[] | 作者节点（AI智能体或人） |
| **ReviewComment** | id, chapterId, content, authorId, createdAt, resolvedAt | 审核意见节点 |

### 3.3 边设计

| 边类型 | 起点 | 终点 | 属性 |
|--------|------|------|------|
| **CONTAINS** | Chapter | Section | order |
| **FOLLOWS** | Chapter/Section | Chapter/Section | logicalType∈{前置,后续,可选} |
| **DEFINES** | Chapter/Section | Concept | definitionText |
| **USES** | Chapter/Section | Term | usageContext |
| **REFERENCES** | Section | Section | referenceType∈{定义引用,数据引用,案例引用}, context |
| **ASSIGNED_TO** | Chapter/Section | Author | assignedAt |
| **REVIEWED_BY** | Chapter/Section | Author | reviewAt, status |
| **SIMILAR_TO** | Concept | Concept | similarityScore |

### 3.4 图谱操作接口

```python
class KnowledgeGraphInterface:
    """知识图谱操作接口"""

    # 章节结构操作
    def create_chapter(self, title: str, order: int) -> ChapterNode:
        """创建章节点"""

    def create_section(self, chapter_id: str, title: str, order: int) -> SectionNode:
        """创建节节点"""

    def link_chapters(self, from_id: str, to_id: str, logical_type: str):
        """链接章节，维护逻辑链"""

    # 知识条目操作
    def add_concept(self, name: str, definition: str, domain: str) -> ConceptNode:
        """添加概念"""

    def add_term(self, term: str, definition: str, domain: str) -> TermNode:
        """添加术语"""

    def link_term_to_concept(self, term_id: str, concept_id: str):
        """建立术语与概念的联系"""

    # 检索操作
    def get_chapter_context(self, chapter_id: str, depth: int = 1) -> dict:
        """获取章节上下文（包含前后续章节）"""

    def find_concept_similar(self, concept_id: str, threshold: float = 0.8) -> list[ConceptNode]:
        """查找相似概念（用于RAG）"""

    def get_term_definitions(self, term: str) -> list[TermNode]:
        """获取术语定义（可能有多个来源）"""
```

## 第四章 多智能体协同架构

### 4.1 智能体类型定义

| 智能体 | 角色 | 输入 | 输出 | 协作方式 |
|--------|------|------|------|----------|

## 第五章 上下文管理设计

### 5.1 上下文预算模型（修订版）

基于《技术调研-上下文管理.md》的分析，Token预算调整为：

| 层级 | 容量 | 内容 | 生命周期 |
|------|------|------|----------|
| **L0 核心上下文** | 60K Tokens | 大纲架构、术语表、风格指南、角色设定、全局逻辑链 | 整个项目 |
| **L1 弹性材料** | 40K Tokens | 当前章节参考内容、RAG检索结果、编写要求 | 当前章节编写 |
| **L2 章节状态** | ~5K Tokens | 章节元数据（进度、版本、问题标记） | 当前章节 |
| **L3 输出缓冲** | 10K Tokens | 响应正文生成 | 单次调用 |
| **总计** | **~115K Tokens/章节** | 预留15%余量 | - |

**预算控制原则**：
1. L0内容固定只读，仅在项目初始化和重大变更时更新
2. L1内容动态加载，每章节编写前根据内容寻址哈希获取
3. L2状态轻量存储，不计入上下文预算
4. 单章节超出预算时，触发RAG检索淘汰最远章节

### 5.2 上下文加载策略

```python
class ContextManager:
    """上下文管理器"""

    def __init__(self, redis_client: Redis, neo4j_driver: Driver):
        self.redis = redis_client
        self.neo4j = neo4j_driver
        self.core_context_key = "context:L0:core"
        self.material_key_template = "context:L1:{chapter_id}"

    async def load_chapter_context(
        self,
        chapter_id: str,
        max_tokens: int = 40000
    ) -> dict:
        """
        加载章节编写所需的完整上下文

        策略：
        1. 加载L0核心上下文（固定）
        2. 加载L1弹性材料（按内容哈希）
        3. RAG检索补充（如预算允许）
        4. 截断至max_tokens限制
        """
        context = {
            "L0": await self._load_core_context(),
            "L1": await self._load_material(chapter_id),
            "chapter_state": await self._load_chapter_state(chapter_id)
        }

        total_tokens = self._estimate_tokens(context)
        if total_tokens > max_tokens:
            context = self._truncate_context(context, max_tokens)

        return context

    async def _load_core_context(self) -> dict:
        """加载L0核心上下文（固定只读）"""
        cached = await self.redis.get(self.core_context_key)
        if cached:
            return json.loads(cached)

        # 从Neo4j加载完整核心上下文
        core = await self._build_core_context_from_graph()
        await self.redis.setex(
            self.core_context_key,
            3600,  # 1小时缓存
            json.dumps(core)
        )
        return core

    async def _load_material(self, chapter_id: str) -> dict:
        """加载L1弹性材料（按章节）"""
        key = self.material_key_template.format(chapter_id=chapter_id)
        cached = await self.redis.get(key)
        if cached:
            return json.loads(cached)

        # 从Neo4j和向量数据库加载相关材料
        material = await self._build_material_from_sources(chapter_id)
        await self.redis.setex(
            key,
            300,  # 5分钟缓存
            json.dumps(material)
        )
        return material
```

### 5.3 内容寻址与缓存

**问题**：Token位置引用在上下文重放时会失效（如模型生成过程中上下文窗口滑动）。

**解决方案**：使用内容寻址哈希替代位置引用。

```python
class ContentAddressableCache:
    """内容寻址缓存"""

    def store(self, content: str, metadata: dict) -> str:
        """
        存储内容并返回内容哈希

        格式: {hash: "sha256[:16]", offset: 0, length: len(content)}
        """
        content_hash = hashlib.sha256(content.encode()).hexdigest()[:16]
        cache_key = f"content:{content_hash}"

        # 存储内容与元数据
        self.redis.hset(cache_key, mapping={
            "content": content,
            "metadata": json.dumps(metadata),
            "created_at": datetime.now().isoformat()
        })

        # 返回引用格式
        return json.dumps({
            "hash": content_hash,
            "offset": 0,
            "length": len(content)
        })

    def retrieve(self, content_ref: str) -> dict | None:
        """
        通过内容引用获取内容

        content_ref: '{"hash": "abc123...", "offset": 0, "length": 500}'
        """
        ref = json.loads(content_ref)
        cache_key = f"content:{ref['hash']}"

        data = self.redis.hgetall(cache_key)
        if not data:
            return None

        return {
            "content": data["content"],
            "metadata": json.loads(data["metadata"])
        }

    def compute_hash(self, content: str) -> str:
        """计算内容哈希（不存储）"""
        return hashlib.sha256(content.encode()).hexdigest()[:16]
```

### 5.4 上下文膨胀控制

**触发条件**：当单章节上下文超过45K Tokens时，触发膨胀控制。

**控制策略**：
1. **RAG淘汰**：优先淘汰最远章节的参考内容（基于FOLLOWS边的距离）
2. **摘要压缩**：对L1材料进行摘要，保留关键信息
3. **历史截断**：仅保留最近3轮的对话历史

```python
async def handle_context_expansion(
    self,
    chapter_id: str,
    current_tokens: int,
    threshold: int = 45000
) -> dict:
    """上下文膨胀处理"""

    if current_tokens <= threshold:
        return {"action": "NONE", "tokens": current_tokens}

    # 策略1：RAG淘汰
    far_chapters = await self._get_far_chapters(chapter_id, distance=3)
    for ch in far_chapters:
        await self._evict_chapter_material(ch.id)

    evicted_tokens = sum(self._estimate_tokens(ch) for ch in far_chapters)
    if current_tokens - evicted_tokens <= threshold:
        return {
            "action": "RAG_EVICT",
            "evicted": [ch.id for ch in far_chapters],
            "tokens": current_tokens - evicted_tokens
        }

    # 策略2：摘要压缩
    compressed = await self._compress_material(chapter_id)
    return {
        "action": "COMPRESS",
        "method": "abstractive_summary",
        "tokens": self._estimate_tokens(compressed)
    }
```

### 6.3 生成质量控制

#### 6.3.1 多轮迭代生成

```python
class IterativeGeneration:
    """迭代式内容生成"""

    def __init__(
        self,
        llm_client,
        quality_threshold: float = 0.8,
        max_iterations: int = 3
    ):
        self.llm = llm_client
        self.quality_threshold = quality_threshold
        self.max_iterations = max_iterations

    async def generate_with_feedback(
        self,
        prompt: str,
        quality_feedback_fn: callable
    ) -> str:
        """
        带质量反馈的迭代生成

        流程：
        1. 生成初稿
        2. 质量评估
        3. 如不达标，收集反馈并重写
        4. 最多迭代max_iterations次
        """
        draft = await self.llm.generate(prompt)
        iteration = 0

        while iteration < self.max_iterations:
            quality_score, feedback = await quality_feedback_fn(draft)

            if quality_score >= self.quality_threshold:
                return draft

            # 将反馈注入Prompt重新生成
            revised_prompt = f"""
{prompt}

## 上一轮生成评估
质量得分：{quality_score}/1.0
不达标原因：{feedback}

## 修订要求
请根据以上反馈修订内容，特别注意：
{feedback}
"""
            draft = await self.llm.generate(revised_prompt)
            iteration += 1

        # 达到最大迭代次数，返回当前版本
        return draft
```

| 类型 | 描述 | 检测策略 | 规避方法 |
|------|------|----------|----------|
| **事实性幻觉** | 与可验证事实矛盾 | 知识库比对、RAG验证 | 引入外部知识检索 |
| **逻辑性幻觉** | 内部逻辑矛盾 | 逻辑一致性检查 | 多跳推理验证 |
| **语义幻觉** | 看似合理但不准确 | 置信度阈值控制 | 引用权威来源 |

### 7.3 检测实现

```python
class HallucinationDetector:
    """幻觉检测器"""

    def __init__(
        self,
        knowledge_graph: KnowledgeGraphInterface,
        external_kb: ExternalKnowledgeAPI,
        llm_for_scoring: LLMClient
    ):
        self.kg = knowledge_graph
        self.external_kb = external_kb
        self.llm = llm_for_scoring

    async def detect(self, content: str, context: dict) -> DetectionResult:
        """
        综合幻觉检测

        返回：{
            "factual_score": float,      # 0-1，越高越好
            "logical_score": float,
            "semantic_score": float,
            "overall_score": float,
            "issues": [
                {"type": "factual", "text": "xxx", "suggestion": "xxx"}
            ]
        }
        """
        # 并行执行三类检测
        factual_task = self._check_factual(content, context)
        logical_task = self._check_logical(content, context)
        semantic_task = self._check_semantic(content, context)

        factual_result, logical_result, semantic_result = await asyncio.gather(
            factual_task, logical_task, semantic_task
        )

        # 加权综合评分
        overall = (
            factual_result["score"] * 0.4 +
            logical_result["score"] * 0.3 +
            semantic_result["score"] * 0.3
        )

        return DetectionResult(
            factual_score=factual_result["score"],
            logical_score=logical_result["score"],
            semantic_score=semantic_result["score"],
            overall_score=overall,
            issues=factual_result["issues"] +
                   logical_result["issues"] +
                   semantic_result["issues"]
        )

    async def _check_factual(self, content: str, context: dict) -> dict:
        """事实性检测：与知识库比对"""
        issues = []
        score = 1.0

        # 提取声称事实（实体、数值、日期等）
        claimed_facts = await self._extract_claims(content)

        for fact in claimed_facts:
            # 查询知识库验证
            verified = await self._verify_fact(fact, context)

            if not verified["verified"]:
                # 查询外部知识库
                external_result = await self.external_kb.lookup(
                    fact["entity"]
                )

                if not external_result["found"]:
                    issues.append({
                        "type": "factual",
                        "text": fact["original_text"],
                        "claim": fact["entity"],
                        "suggestion": f"无法验证此事实，请提供权威来源或删除"
                    })
                    score -= 0.1

        return {"score": max(0, score), "issues": issues}

    async def _check_logical(self, content: str, context: dict) -> dict:
        """逻辑性检测：推理链一致性"""
        issues = []
        score = 1.0

        # 提取因果链条
        causal_chains = await self._extract_causal_chains(content)

        for chain in causal_chains:
            # 验证因果关系是否合理
            is_valid = await self._validate_causal_chain(chain)

            if not is_valid:
                issues.append({
                    "type": "logical",
                    "text": chain["summary"],
                    "suggestion": "因果关系存疑，请重新推导"
                })
                score -= 0.15

        return {"score": max(0, score), "issues": issues}

    async def _check_semantic(self, content: str, context: dict) -> dict:
        """语义置信度检测：多次采样"""
        issues = []

        # 高温采样多次，检查一致性
        samples = await self.llm.generate_multiple(
            prompt=f"请判断以下内容中每个陈述的可信度（1-5分）：\n{content}",
            temperature=0.9,
            n=5
        )

        # 分析采样一致性
        consistency = self._analyze_sampling_consistency(samples)

        if consistency < 0.7:
            low_confidence_paras = self._find_low_confidence_paras(samples)
            for para in low_confidence_paras:
                issues.append({
                    "type": "semantic",
                    "text": para,
                    "suggestion": "此段落置信度较低，建议核实或补充依据"
                })

        return {
            "score": consistency,
            "issues": issues
        }
```

### 7.4 规避策略集成

将幻觉规避策略嵌入生成流程：

```python
class HallucinationAwareGenerator:
    """具备幻觉规避能力的生成器"""

    def __init__(
        self,
        base_generator: IterativeGeneration,
        detector: HallucinationDetector,
        retriever: KnowledgeRetriever
    ):
        self.generator = base_generator
        self.detector = detector
        self.retriever = retriever

    async def generate_with_avoidance(
        self,
        prompt: str,
        context: dict
    ) -> GenerationResult:
        """
        带幻觉规避的生成流程

        1. 生成初稿
        2. 幻觉检测
        3. 针对问题补充RAG依据
        4. 重写不通过段落
        5. 验证通过后输出
        """

        # Step 1: 生成初稿
        draft = await self.generator.generate_with_feedback(
            prompt,
            self._quality_feedback
        )

        # Step 2: 幻觉检测
        detection = await self.detector.detect(draft, context)

        # Step 3: RAG补充依据
        if detection.issues:
            for issue in detection.issues:
                if issue["type"] == "factual":
                    # RAG检索补充依据
                    evidence = await self.retriever.retrieve(
                        query=issue["claim"],
                        top_k=3
                    )
                    issue["supporting_evidence"] = evidence

        # Step 4: 重写不通过段落
        if detection.overall_score < 0.8:
            draft = await self._revise_with_evidence(draft, detection)

        # Step 5: 最终验证
        final_detection = await self.detector.detect(draft, context)

        return GenerationResult(
            content=draft,
            detection=final_detection,
            passed=final_detection.overall_score >= 0.8
        )
```

### 8.2 关键合规节点

#### 8.2.1 版权合规

**阈值标准（修订版）**：
- **连续引用**：超过**300字符**连续引用同一来源，需标注来源并获授权
- **相似度**：文本相似度超过**10%**触发进一步审核
- **原创性要求**：全书原创性内容占比≥70%

**技术实现**：

```python
class CopyrightCompliance:
    """版权合规检查"""

    def __init__(self, similarity_checker: TextSimilarityChecker):
        self.checker = similarity_checker

    async def check_copyright(
        self,
        content: str,
        context: dict
    ) -> ComplianceResult:
        """
        版权合规检查

        检查项：
        1. 连续引用长度检测
        2. 文本相似度检测
        3. 来源标注完整性
        """
        issues = []

        # 1. 连续引用检测
        continuous_quotes = self._find_continuous_quotes(content)
        for quote in continuous_quotes:
            if quote["length"] > 300:
                issues.append({
                    "type": "continuous_quote",
                    "position": quote["position"],
                    "length": quote["length"],
                    "severity": "HIGH",
                    "message": f"连续引用超过300字符（实际{quote['length']}字符），需获授权",
                    "action": "REQUIRE_AUTHORIZATION"
                })

        # 2. 相似度检测
        similarity_result = await self.checker.check_similarity(
            content,
            threshold=0.10  # 10%阈值
        )

        if similarity_result["overall_similarity"] > 0.10:
            issues.append({
                "type": "high_similarity",
                "similar_sources": similarity_result["similar_sources"],
                "severity": "MEDIUM",
                "message": "相似度超过10%，需人工审核",
                "action": "REQUIRE_HUMAN_REVIEW"
            })

        # 3. 来源标注检查
        missing_citations = self._check_citation_completeness(content)
        if missing_citations:
            issues.extend(missing_citations)

        return ComplianceResult(
            passed=len([i for i in issues if i["severity"] == "HIGH"]) == 0,
            issues=issues,
            risk_level=self._calculate_risk_level(issues)
        )
```

### 10.2 自动化评测方案

```python
class EvaluationSystem:
    """评测系统"""

    def __init__(
        self,
        llm_judge: LLMClient,
        similarity_checker: TextSimilarityChecker,
        terminology_db: TerminologyDB
    ):
        self.llm = llm_judge
        self.similarity = similarity_checker
        self.terminology = terminology_db

    async def evaluate_chapter(
        self,
        chapter: Chapter,
        ground_truth: dict | None = None
    ) -> EvaluationResult:
        """
        章节评测

        返回综合评测报告
        """
        report = EvaluationReport(chapter_id=chapter.id)

        # 1. 术语一致性评测
        term_consistency = await self._evaluate_term_consistency(chapter)
        report.add_metric("term_consistency", term_consistency)

        # 2. 风格一致性评测
        style_consistency = await self._evaluate_style_consistency(chapter)
        report.add_metric("style_consistency", style_consistency)

        # 3. 格式规范评测
        format_compliance = await self._evaluate_format_compliance(chapter)
        report.add_metric("format_compliance", format_compliance)

        # 4. 原创性评测
        originality = await self._evaluate_originality(chapter)
        report.add_metric("originality", originality)

        # 5. 知识准确性评测（幻觉检测）
        accuracy = await self._evaluate_factual_accuracy(chapter)
        report.add_metric("factual_accuracy", accuracy)

        # 6. 可读性评测
        readability = await self._evaluate_readability(chapter)
        report.add_metric("readability", readability)

        # 综合评分
        report.overall_score = (
            term_consistency * 0.15 +
            style_consistency * 0.15 +
            format_compliance * 0.10 +
            originality * 0.15 +
            accuracy * 0.30 +
            readability * 0.15
        )

        return report

    async def _evaluate_term_consistency(
        self,
        chapter: Chapter
    ) -> float:
        """术语一致性评测"""
        expected_terms = self.terminology.get_for_chapter(chapter.id)
        used_terms = chapter.extract_terms()

        if not expected_terms:
            return 1.0  # 无术语要求则通过

        matched = sum(1 for t in used_terms if t in expected_terms)
        return matched / len(expected_terms) if expected_terms else 1.0
```
