### 1.2 知识图谱的三大核心作用

| 作用 | 具体场景 | 用户价值 |
|------|---------|---------|
| **外部记忆** | 跨章节依赖查询、术语定义追溯 | 保持全书一致性 |
| **逻辑验证器** | 依赖冲突检测、循环引用检测 | 提前发现逻辑错误 |
| **引用解析器** | [@ref:xxx]令牌定位、引用完整性验证 | 快速定位引用内容 |

### 1.3 设计约束

| 约束项 | 说明 |
|--------|------|
| **更新频率** | 章节完成后批量更新，非实时 |
| **查询延迟** | <100ms（P99）用于写作时的术语补全 |
| **一致性强要求** | 图谱与frontmatter双写一致性 |
| **可解释性** | 所有关系可追溯到源内容 |

## 2. 知识图谱架构设计

### 2.1 混合粒度节点设计

采用**章节+概念双层节点**设计，兼顾粗细粒度优势：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           节点层次设计                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  L1 章节节点（Chapter）                                                    │
│  ├── 粒度：章为单位                                                       │
│  ├── 属性：chapter_id, title, version, status, author                      │
│  └── 用途：依赖关系管理、进度追踪                                          │
│                                                                             │
│  L2 节节点（Section）                                                      │
│  ├── 粒度：节为单位（1.1, 1.2, 2.1等）                                    │
│  ├── 属性：section_id, title, chapter_id, order                           │
│  └── 用途：引用关系定位、细粒度依赖                                        │
│                                                                             │
│  L3 概念节点（Concept）                                                     │
│  ├── 粒度：核心概念/术语                                                   │
│  ├── 属性：concept_id, term, definition, defined_at, aliases               │
│  └── 用途：术语一致性校验、跨章节引用                                       │
│                                                                             │
│  L4 论点节点（Argument）                                                    │
│  ├── 粒度：核心论点（每节1-3个）                                          │
│  ├── 属性：argument_id, claim, chapter_id, section_id, strength           │
│  └── 用途：论点支撑关系、逻辑链构建                                        │
│                                                                             │
│  L5 证据节点（Evidence）                                                    │
│  ├── 粒度：独立证据单元                                                   │
│  ├── 属性：evidence_id, type, content_hash, source, credibility           │
│  └── 用途：论点支撑、引用验证                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.5 存储选型对比

| 维度 | Neo4j | PostgreSQL + pgRouting | RedisGraph | 选型结论 |
|------|-------|------------------------|------------|---------|
| **图查询性能** | ⭐⭐⭐⭐⭐ 原生图 | ⭐⭐⭐ 需要扩展 | ⭐⭐⭐⭐ 原生图 | Neo4j最优 |
| **关系遍历** | ⭐⭐⭐⭐⭐ 优化 | ⭐⭐ 一般 | ⭐⭐⭐⭐ 优化 | Neo4j/RedisGraph |
| **ACID事务** | ⭐⭐⭐⭐ 原生支持 | ⭐⭐⭐⭐⭐ 原生支持 | ⭐⭐⭐ 较弱 | PostgreSQL最优 |
| **水平扩展** | ⭐⭐⭐ Enterprise | ⭐⭐⭐⭐ 原生支持 | ⭐⭐⭐ 较弱 | PostgreSQL最优 |
| **运维复杂度** | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐⭐ 低 | ⭐⭐⭐ 中等 | PostgreSQL最低 |
| **生态系统** | ⭐⭐⭐⭐ 成熟 | ⭐⭐⭐⭐⭐ 成熟 | ⭐⭐⭐ 较小 | PostgreSQL最优 |
| **与现有系统集成** | ⭐⭐⭐ 需适配 | ⭐⭐⭐⭐⭐ 同一数据库 | ⭐⭐ 需适配 | PostgreSQL最优 |

**推荐方案：PostgreSQL + apoc扩展（推荐）**

理由：
1. 教材系统已有PostgreSQL，简化运维
2. 依赖关系遍历深度有限（一般<5层），pgRouting足够
3. 与frontmatter元数据共存同一数据库，一致性好
4. 备份恢复简单

**备选方案：若图遍历性能成为瓶颈，再引入Neo4j**

# 伪代码：提纲解析
def parse_outline(outline: OutlineParserInput) -> GraphProposal:
    proposal = GraphProposal()

    # 1. 创建章节节点
    for chapter in outline.chapters:
        chapter_node = ChapterNode(
            chapter_id=chapter.id,
            title=chapter.title,
            status=ChapterStatus.DRAFT,
            order=chapter.order,
            part=chapter.part
        )
        proposal.add_node(chapter_node)

        # 2. 创建节节点
        for section in chapter.sections:
            section_node = SectionNode(
                section_id=section.id,
                title=section.title,
                chapter_id=chapter.id,
                order=section.order
            )
            proposal.add_node(section_node)

        # 3. 创建章节依赖关系
        for dep_chapter_id in chapter.dependencies:
            relation = ChapterDependsOnRelation(
                source_chapter_id=chapter.id,
                target_chapter_id=dep_chapter_id,
                dependency_type='HARD' if is_critical_dependency(chapter, dep_chapter_id) else 'SOFT'
            )
            proposal.add_relation(relation)

    # 4. 验证依赖链完整性
    validate_dependency_chain(proposal)

    return proposal
```

#### 3.2.3 循环依赖检测

```python
def detect_circular_dependencies(proposal: GraphProposal) -> List[CircularDependency]:
    """
    使用DFS检测循环依赖
    返回所有循环路径
    """
    graph = build_adjacency_list(proposal.relations)
    cycles = []
    visited = set()
    rec_stack = set()
    path = []

    def dfs(node: str) -> bool:
        visited.add(node)
        rec_stack.add(node)
        path.append(node)

        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                if dfs(neighbor):
                    return True
            elif neighbor in rec_stack:
                # 发现循环
                cycle_start = path.index(neighbor)
                cycles.append(path[cycle_start:] + [neighbor])
                return True

        path.pop()
        rec_stack.remove(node)
        return False

    for node in graph.keys():
        if node not in visited:
            dfs(node)

    return cycles
```

### 3.3 Phase 2: 写作内容分析

#### 3.3.1 LLM实体抽取提示词

```markdown

你是一个教材结构分析专家。请从以下章节内容中抽取知识图谱节点和关系。

## 抽取要求

### 节点类型

1. **概念（Concept）**：教材中首次定义的术语/概念

1. **REFERENCES**：论点引用证据

#### 3.3.2 增量更新策略

```python
class IncrementalGraphUpdater:
    """
    增量更新策略：只更新变更部分
    """

    def __init__(self, graph_store: GraphStore):
        self.graph_store = graph_store
        self.change_detector = ContentChangeDetector()

    def update_chapter(self, chapter_id: str, new_content: str, old_content: str):
        # 1. 检测内容变更
        changes = self.change_detector.detect(old_content, new_content)

        # 2. 计算变更影响域
        affected_nodes = self._get_affected_nodes(chapter_id, changes)
        affected_relations = self._get_affected_relations(affected_nodes)

        # 3. 事务性更新
        with self.graph_store.transaction():
            # 删除受影响的旧节点和关系
            self._remove_obsolete(affected_nodes, affected_relations)

            # 重新抽取并插入新节点和关系
            new_proposal = self._extract_entities(chapter_id, new_content)
            self._insert_new(new_proposal)

        # 4. 触发一致性检查
        self._validate_consistency(chapter_id)

    def _get_affected_nodes(self, chapter_id: str, changes: ContentChanges):
        """
        确定哪些节点受影响
        - 概念定义位置变化
        - 论点内容变化
        - 新增/删除的段落
        """
        affected = set()

        if changes.definition_locations_changed:
            affected.add(f"concept_at_{chapter_id}")

        if changes.arguments_changed:
            # 获取该章节所有论点
            affected.update(self.graph_store.get_arguments_by_chapter(chapter_id))

        return affected
```

async def autocomplete_terms(prefix: str, limit: int = 5) -> List[TermSuggestion]:
    """
    写作时的术语自动补全
    目标延迟：<100ms P99
    """
    # 1. 优先查Redis缓存
    cache_key = f"term:autocomplete:{prefix.lower()}"
    cached = await redis.get(cache_key)

    if cached:
        return json.loads(cached)

    # 2. 缓存未命中，查PostgreSQL
    query = """
        SELECT concept_id, term, category, definition
        FROM concepts
        WHERE term_lower LIKE :prefix || '%'
           OR EXISTS (
               SELECT 1 FROM unnest(aliases) a
               WHERE lower(a) LIKE :prefix || '%'
           )
        ORDER BY
            CASE WHEN term_lower = :prefix THEN 0  -- 精确前缀优先
                 WHEN term_lower LIKE :prefix || '%' THEN 1
                 ELSE 2
            END,
            term
        LIMIT :limit;
    """

    results = await db.fetch_all(query, {
        'prefix': prefix.lower(),
        'limit': limit
    })

    suggestions = [
        TermSuggestion(
            concept_id=r['concept_id'],
            term=r['term'],
            category=r['category'],
            definition_preview=r['definition'][:50] + '...'
        )
        for r in results
    ]

    # 3. 写入缓存（TTL=5分钟）
    await redis.setex(cache_key, 300, json.dumps(suggestions))

    return suggestions

async def validate_dependency_chain(chapter_id: str) -> ValidationResult:
    """
    验证章节依赖链完整性
    用于提纲审核和写作前检查
    """
    result = ValidationResult()

    # 1. 获取所有依赖
    query = """
        SELECT target_chapter_id, dependency_type, verified
        FROM chapter_depends_on
        WHERE source_chapter_id = :chapter_id;
    """
    dependencies = await db.fetch_all(query, {'chapter_id': chapter_id})

    # 2. 检查被依赖章节是否存在
    for dep in dependencies:
        chapter_exists = await db.fetch_one(
            "SELECT 1 FROM chapters WHERE chapter_id = :id",
            {'id': dep['target_chapter_id']}
        )

        if not chapter_exists:
            result.add_error(
                f"依赖的章节 {dep['target_chapter_id']} 不存在",
                severity='HIGH'
            )
        elif dep['dependency_type'] == 'HARD' and not dep['verified']:

                f"硬依赖 {dep['target_chapter_id']} 未经验证",
                severity='MEDIUM'

            f"检测到循环依赖: {' -> '.join(circular[0])}",
            severity='CRITICAL'
        )

    # 4. 验证依赖传递闭包
    all_deps = await get_transitive_dependencies(chapter_id)
    result.transitive_dependencies = all_deps

    return result

async def resolve_evidence_reference(ev_id: str) -> ReferenceResult:
    """
    解析证据引用

    query = """
        SELECT e.*, a.chapter_id, a.section_id

        found=True,
        type='evidence',
        content={
            'evidence_id': result['evidence_id'],

async def detect_contradictions(chapter_ids: List[str] = None) -> List[Contradiction]:
    """
    检测论点间的矛盾关系
    使用LLM进行语义层面矛盾检测
    """
    # 1. 获取待检测论点
    if chapter_ids:
        query = """
            SELECT argument_id, claim, chapter_id, section_id
            FROM arguments
            WHERE chapter_id = ANY(:chapter_ids);
        """
    else:
        query = """
            SELECT argument_id, claim, chapter_id, section_id
            FROM arguments;
        """

    arguments = await db.fetch_all(query, {'chapter_ids': chapter_ids})

    # 2. 批量构建矛盾检测任务
    detection_tasks = []
    for i, arg1 in enumerate(arguments):
        for arg2 in arguments[i+1:]:
            # 只检测同一章节内的论点对
            if arg1['chapter_id'] != arg2['chapter_id']:
                continue

            detection_tasks.append({
                'argument_pair': (arg1, arg2),
                'prompt': build_contradiction_prompt(arg1, arg2)
            })

    # 3. 并行执行矛盾检测（可限制并发）
    contradictions = []
    for batch in chunked(detection_tasks, batch_size=10):
        results = await llm_batch_evaluate(batch)

        for task, result in zip(batch, results):
            if result.is_contradictory:
                contradictions.append(Contradiction(
                    argument1=task['argument_pair'][0],
                    argument2=task['argument_pair'][1],
                    conflict_type=result.conflict_type,
                    severity=result.severity,
                    evidence=result.evidence
                ))

    # 4. 存储矛盾关系
    await store_contradictions(contradictions)

    return contradictions
```

```python
class HybridRetrievalService:
    """
    混合检索：知识图谱 + 向量检索
    """

    async def retrieve_for_writing(
        self,
        query: str,
        context: WritingContext,
        top_k: int = 10
    ) -> RetrievalResult:
        """
        写作时的混合检索
        """
        results = []

        # 1. 知识图谱结构化查询
        if context.current_chapter:
            # 获取章节依赖信息
            dependencies = await self.graph.get_direct_dependencies(
                context.current_chapter.chapter_id
            )

            # 获取本章已定义概念
            defined_concepts = await self.graph.get_defined_concepts(
                context.current_chapter.chapter_id
            )

            # 获取待引用概念
            pending_concepts = await self.graph.get_pending_concepts(
                context.current_chapter.chapter_id
            )

            results.extend([
                StructuredResult(
                    type='dependency',
                    data=dependencies
                ),
                StructuredResult(
                    type='defined_concept',
                    data=defined_concepts
                ),
                StructuredResult(
                    type='pending_concept',
                    data=pending_concepts
                )
            ])

        # 2. 向量语义检索
        semantic_results = await self.vector_store.search(
            query=query,
            top_k=top_k,
            filters={'chapter_id': context.current_chapter.chapter_id}
        )

        # 3. 混合排序
        reranked = await self.reranker.rerank(
            query=query,
            results=results + semantic_results,
            weights={'structured': 0.4, 'semantic': 0.6}
        )

        return reranked[:top_k]
```

```python
class LogicChainSynchronizer:
    """
    知识图谱 ↔ 逻辑链文档同步
    """

    def __init__(self, graph_store, logic_chain_store):
        self.graph = graph_store
        self.logic_chain = logic_chain_store

    async def sync_chapter_to_logic_chain(self, chapter_id: str):
        """
        将章节图谱变更同步到逻辑链文档
        """
        # 1. 获取章节的论点
        arguments = await self.graph.get_arguments_by_chapter(chapter_id)

        # 2. 构建论点链
        argument_chain = []
        for arg in arguments:
            supporting_evidence = await self.graph.get_supporting_evidence(arg.id)

            argument_chain.append({
                'chapter': chapter_id,
                'argument_id': arg.id,
                'claim': arg.claim,
                'strength': arg.strength,
                'evidence': [e.content_hash for e in supporting_evidence],
                'logical_role': arg.logical_role
            })

        # 3. 更新逻辑链文档
        await self.logic_chain.update_chapter_arguments(chapter_id, argument_chain)

        # 4. 检测是否需要更新全局逻辑链
        if await self._is_global_impact(chapter_id):
            await self._update_global_logic_chain(chapter_id)

    async def sync_logic_chain_to_graph(self, logic_chain_doc: LogicChainDoc):
        """
        将逻辑链文档变更同步到图谱（用于人工编辑逻辑链）
        """
        # 1. 解析逻辑链文档变更
        changes = self.logic_chain.diff(logic_chain_doc)

        # 2. 应用变更到图谱
        async with self.graph.transaction():
            for change in changes:
                if change.type == 'ARGUMENT_MODIFIED':
                    await self.graph.update_argument(
                        change.argument_id,
                        change.new_data
                    )
                elif change.type == 'RELATION_MODIFIED':
                    await self.graph.update_relation(
                        change.relation_id,
                        change.new_data
                    )

        # 3. 触发一致性检查
        await self._validate_consistency()
```

        """

        # 1. 更新/创建概念节点
        concept = ConceptNode(
            concept_id=term.term_id,
            term=term.term,
            term_en=term.term_en,
            definition=term.definition.short,
            category=term.category,
            defined_at={
                'chapter_id': term.defined_in.chapter,
                'section_id': term.defined_in.section
            }
        )

        await self.graph.upsert_node(concept)

        # 2. 重建概念关系
        for related_id in term.related_terms:
            await self.graph.add_relation(
                'CONCEPT_RELATED',
                source=term.term_id,
                target=related_id
            )

        # 3. 验证术语一致性
        conflicts = await self._detect_glossary_conflicts(term)
        if conflicts:

### 6.2 图遍历查询示例

```sql
-- 查询章节依赖链（递归）
WITH RECURSIVE dependency_chain AS (
    -- 基例：直接依赖
    SELECT
        source_chapter,
        target_chapter,
        dependency_type,
        1 as depth,
        ARRAY[source_chapter] as path
    FROM chapter_depends_on
    WHERE source_chapter = 'ch03'

    UNION ALL

    -- 递归：依赖的依赖
    SELECT
        d.source_chapter,
        d.target_chapter,
        d.dependency_type,
        dc.depth + 1,
        dc.path || d.source_chapter
    FROM chapter_depends_on d
    JOIN dependency_chain dc ON d.source_chapter = dc.target_chapter
    WHERE NOT d.target_chapter = ANY(dc.path)  -- 避免循环
      AND dc.depth < 10  -- 限制深度
)
SELECT * FROM dependency_chain ORDER BY depth;

-- 检测循环依赖
WITH RECURSIVE cycle AS (
    SELECT
        source_chapter,
        target_chapter,
        ARRAY[source_chapter] as path,
        FALSE as is_cycle
    FROM chapter_depends_on

    UNION ALL

    SELECT
        d.source_chapter,
        d.target_chapter,
        c.path || d.source_chapter,
        d.target_chapter = ANY(c.path)
    FROM chapter_depends_on d
    JOIN cycle c ON d.source_chapter = c.target_chapter
    WHERE NOT c.is_cycle
)
SELECT * FROM cycle WHERE is_cycle = TRUE;

-- 概念依赖路径
WITH RECURSIVE concept_prereq AS (
    SELECT
        parent_concept,
        child_concept,
        1 as order_num,
        ARRAY[parent_concept] as path
    FROM concept_hierarchy
    WHERE relation_type = 'prerequisite_of'

    UNION ALL

    SELECT
        ch.parent_concept,
        ch.child_concept,
        cp.order_num + 1,
        cp.path || ch.parent_concept
    FROM concept_hierarchy ch
    JOIN concept_prereq cp ON ch.child_concept = cp.parent_concept
    WHERE NOT ch.parent_concept = ANY(cp.path)
)
SELECT * FROM concept_prereq ORDER BY order_num;
