```
                      /───────\
                     / 集成测试 \           ← 模块间交互、Agent协作
```

### 2.2 L1 语法格式检测

```yaml
L1_checklist:
  markdown_format:
    - 标题层级连续性（H1→H2→H3）
    - 列表项一致性（- 或 * 或数字）
    - 代码块语法正确性
    - 链接格式有效性
    
  citation_format:
    - DOI格式: 10.\d{4,}/[-._;()/:A-Z0-9]+
    - 引用标记位置正确（句末或段末）
    - 多次引用同一来源的标记一致性
    
  word_count:
    - 单章节字数偏差 ±10%
    - 全书总字数偏差 ±5%
    - 段落平均长度 100-500字
```

**自动化检测脚本**：

```python
class L1FormatValidator:
    """L1语法格式验证器"""
    
    def validate_markdown(self, content: str) -> ValidationResult:
        issues = []
        issues.extend(self._check_heading_hierarchy(content))
        issues.extend(self._check_list_consistency(content))
        issues.extend(self._check_code_blocks(content))
        return ValidationResult(passed=len(issues)==0, issues=issues)
    
    def validate_citation_format(self, content: str) -> ValidationResult:
        dois = self._extract_dois(content)
        for doi in dois:
            if not self._is_valid_doi_format(doi):
                issues.append(f"Invalid DOI format: {doi}")
        return ValidationResult(passed=len(issues)==0, issues=issues)
```

### 2.3 L2 内容一致性检测

```yaml
L2_checklist:
  terminology_consistency:
    - 与全局术语表对比（精确匹配）
    - 同一概念全书的表述一致性
    - 专业术语首次定义位置
    
  reference_validity:
    - 引用的章节/图表/公式存在
    - 引用编号连续性
    - 前向引用（非循环引用）
    
  logical_coherence:
    - 段落间过渡自然
    - 因果关系成立
    - 归纳/演绎推理完整
```

**评分标准**：

| 维度 | 权重 | 评分方法 | 通过阈值 |
|------|------|----------|----------|
| 术语一致性 | 30% | 术语表匹配率 | ≥95% |
| 引用有效性 | 40% | 引用目标存在率 | 100% |
| 逻辑连贯 | 30% | LLMJudge评分 | ≥7/10 |

### 2.4 L3 知识正确性检测

```yaml
L3_checklist:
  doi_verification:
    - DOI存在性验证（CrossRef API）
    - 元数据一致性（标题、年份、作者）
    - 引用内容与原文相关性
    
  regulation_check:
    - 法规编号真实性
    - 法规有效性（是否已被废止）
    - 引用条文与原文一致性
    
  statistics_check:
    - 数据来源可追溯
    - 数值范围合理性
    - 时间戳有效性
```

**API集成**：

```python
class L3KnowledgeValidator:
    """L3知识正确性验证器"""
    
    async def verify_doi(self, doi: str, claimed_metadata: dict) -> VerificationResult:
        """DOI完整验证流程"""
        # 1. CrossRef API查询
        metadata = await self.crossref.get_works(doi)
        
        # 2. 元数据一致性检查
        issues = []
        if claimed_metadata.get('title') not in metadata['title']:
            issues.append(f"Title mismatch")
        if claimed_metadata.get('year') != metadata['year']:
            issues.append(f"Year mismatch")
            
        return VerificationResult(
            valid=len(issues)==0,
            confidence='high' if not issues else 'low',
            issues=issues
        )
```

## 3. Golden Dataset设计

### 3.1 设计原则

```yaml
样本规模: 不少于10个测试样本
质量分布:
  - 高质量样本: 2-3个 (9-10分)
  - 中等质量样本: 4-5个 (7-8分)
  - 低质量样本: 2-3个 (3-4分)
  - 边界case: 2-3个 (含幻觉注入)

每样本结构:
  - id: 唯一标识
  - type: 质量分类
  - content: 教材内容文本
  - expected_score: 预期综合评分
  - key_features: 关键特征列表
  - known_issues: 已知问题（用于检测能力验证）
  - metadata: 元数据（学科、难度、字数等）
```

## 4. 混沌测试场景

### 4.1 设计原则

```yaml
故障注入策略:
  - 随机注入：在正常测试中随机触发故障
  - 定向注入：针对特定场景注入特定故障
  - 渐进注入：从轻度故障逐步升级到严重故障

监控指标:
  - 可用性：系统是否可响应
  - 一致性：数据是否损坏
  - 降级行为：功能是否按预期降级
```
